# /home/kwang/work/skills

Cross-platform AI agent skills for Claude Code and OpenCode, following the
[Agent Skills Specification](https://agentskills.io).

This repository is the **single source of truth** for every skill the user
authors. Skills are written here, then symlinked (or copied) into the
appropriate agent install path.

The first skill in this repo is [`create-skill/`](create-skill/SKILL.md) - a
meta-skill that produces other skills. It works **with or without Python**:
the agent's built-in `bash` / `read` / `write` / `edit` tools are sufficient
to run the full workflow. Python scripts under `create-skill/scripts/` are
optional accelerators.

---

## Quick Start

### A. Bare environment (no Python, no Node)

The calling agent reads
[`create-skill/SKILL.md`](create-skill/SKILL.md) and follows the manual path
for each of the 6 stages. The agent uses `bash` to make directories, `read`
to load templates, simple string substitution to fill placeholders, and
`write` / `edit` to commit the result. No external runtime is touched.

Trigger the agent with any of:

- "Create a skill that ..."
- "Scaffold a new skill for ..."
- "Improve the description of `<skill>`"
- "Validate my skill before publishing"

### B. With Python 3.10+ available

The same workflow runs faster via the accelerator scripts.

#### Create a new skill

```bash
python create-skill/scripts/init_skill.py rate-limiter \
    --description "Use this skill whenever the user wants to add rate limiting to an HTTP API." \
    --author "kwang" \
    --role DEV \
    --domain api
```

Or interactively:

```bash
python create-skill/scripts/init_skill.py --interactive
```

This creates `./rate-limiter/` with `SKILL.md`, `evals/evals.json`, and empty
`references/`, `scripts/`, `assets/` directories ready to populate.

#### Validate

```bash
python create-skill/scripts/validate_skill.py rate-limiter
```

Exits 0 on pass. The validator checks frontmatter shape, name/directory
agreement, SemVer, tag taxonomy, and dependency string syntax.

#### Run evals

The eval loop is always agent-driven (the script only aggregates results).
See [`create-skill/SKILL.md`](create-skill/SKILL.md) stage 5 and
[`create-skill/references/eval-methodology.md`](create-skill/references/eval-methodology.md).

```bash
python create-skill/scripts/run_eval.py rate-limiter --latest --split
```

#### Install

```bash
python create-skill/scripts/install_to_platform.py rate-limiter \
    --target opencode \
    --target claude-strict
```

Default is symlink; the source in this repo remains authoritative.

## Conventions

- **Frontmatter required fields**: `name`, `description`, `version`, `author`,
  `license`, `tags`, `requires`, `related`.
- **Tag taxonomy**:
  - Role (UPPERCASE, >=1 required): `DEV`, `QA`, `BA`, `DEVOPS`
  - Domain (lowercase, >=1 required): `api`, `database`, `backend`, `infra`,
    `auth`, `cli`, `data`, `ml`, `meta`
  - Layer / Maturity / Free-form: optional, all lowercase
- **Dependency syntax**: pip / PEP 440 subset
  (`name`, `name ==1.0.0`, `name >=1.0.0`, `name ~=1.0.0`).
- **`requires` vs `related`**: hard (validator fails if missing) vs soft (see
  also).
- **Size limits**: `SKILL.md` body <= 500 lines; description <= 1024 chars.

Full reference: [`create-skill/references/frontmatter-spec.md`](create-skill/references/frontmatter-spec.md).

## Platforms Supported

| Platform | Install target | Notes |
|---|---|---|
| OpenCode (user)   | `--target opencode`         | `~/.config/opencode/skills/` |
| OpenCode (project)| `--target opencode-project` | `./.opencode/skills/` |
| Claude Code (user, strict) | `--target claude-strict` | `~/.claude/skills/`; frontmatter rewritten so `version`/`author`/`tags`/`requires`/`related` nest under `metadata`. Required because Anthropic's `quick_validate.py` rejects unknown top-level keys. |
| Claude Code (user, literal) | `--target claude`       | `~/.claude/skills/`; literal copy. Use only if you accept Anthropic validator warnings. |
| Claude Code (project) | `--target claude-project` | `./.claude/skills/`; literal copy. |

In a bare environment without Python, the equivalent of each target is a
plain `ln -s` (symlink) or `cp -r` (copy). For `claude-strict`, the
frontmatter is rewritten by hand; see
[`create-skill/references/platform-differences.md`](create-skill/references/platform-differences.md).

## Why a meta-skill?

The first skill in this repo is `create-skill` - a skill that produces other
skills. It encodes the 6-stage workflow (intent -> interview -> scaffold ->
test cases -> eval -> iterate) and is itself subject to the same validation
and eval rules as every other skill. Dogfooding the toolchain on the
toolchain catches gaps early.

Because the workflow is documented in `SKILL.md` itself (not only in the
scripts), this repo works equally well for users who have no Python /
Node toolchain installed. The scripts are accelerators, not prerequisites.
