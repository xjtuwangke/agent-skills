## [1.2.0] - 2024-05-14

### Added
- Public skill template (`templates/skill-template.md`) with canonical structure.
- **Pre-execution Check** section is now MANDATORY for all generated skills.
- **Safety Boundaries** section is now MANDATORY with Forbidden Operations,
  Confirmation Gates, and Emergency Stop.
- Added Pre-execution Check and Safety Boundaries to create-skill itself.
- Hard Rules expanded: Rules 13-15 cover pre-check, safety boundaries, and
  confirmation gates.
- Verification Checklist expanded to verify pre-check and safety sections.

## [1.1.0] - 2024-05-14

### Added
- `suggests` frontmatter field for optional dependencies (tools, runtimes, mcps).
- Documented `suggests` in frontmatter-spec.md with examples.
- Added `suggests.runtimes: [python >=3.10]` to create-skill frontmatter.
- Validator now checks `suggests` structure (unknown keys warned, lists enforced).
- Template updated to include `suggests` block.

### Changed
- Hard Rules expanded to cover `suggests` usage.
- Common Pitfalls added: do not put required runtimes in `suggests`.
- **Hard Rule 12 added: ALL output MUST be in English.**

## [1.0.0] - 2024-05-14

### Added
- Initial release of create-skill meta-skill.
- Dual-path workflow: manual (agent tools only) + optional Python accelerators.
- Frontmatter spec with version, author, tags, requires/related dependencies.
- Five accelerator scripts: init_skill, validate_skill, run_eval, improve_description, install_to_platform.
- Four reference documents: frontmatter-spec, platform-differences, description-patterns, eval-methodology.
- Templates for SKILL.md and evals.json.
- Pre-publish checklist asset.
- Self-validation: validator passes on create-skill itself.
