# Migration Naming Conventions

Reference document for file naming patterns used by different migration
frameworks. The `database-migration` skill auto-detects these patterns.

---

## Alembic (SQLAlchemy)

```
alembic/versions/
├── 20240115_120000_add_users_table.py
├── 20240115_123000_add_email_column.py
└── 20240115_130000_create_orders_table.py
```

- Format: `<timestamp>_<description>.py` or `<revision>_<description>.py`
- Generated by: `alembic revision -m "description"`
- Registry: `alembic_version` table in database

## Knex.js

```
migrations/
├── 20240115120000_add_users_table.js
├── 20240115123000_add_email_column.js
└── 20240115130000_create_orders_table.js
```

- Format: `<timestamp>_<description>.js`
- Generated by: `knex migrate:make description`
- Registry: `knex_migrations` table in database

## Prisma

```
prisma/migrations/
├── 20240115120000_add_users_table/
│   └── migration.sql
├── 20240115123000_add_email_column/
│   └── migration.sql
└── migration_lock.toml
```

- Format: `<timestamp>_<description>/migration.sql`
- Generated by: `prisma migrate dev --name description`
- Registry: `_prisma_migrations` table in database

## Django

```
myapp/migrations/
├── 0001_initial.py
├── 0002_add_user_email.py
└── 0003_create_order_model.py
```

- Format: `<4_digit_sequence>_<description>.py`
- Generated by: `python manage.py makemigrations`
- Registry: `django_migrations` table in database

---

## Detection Priority

The skill checks in this order:

1. `alembic.ini` → Alembic
2. `knexfile.js` → Knex
3. `prisma/migrations/` → Prisma
4. `manage.py` + `*/migrations/` → Django
5. None detected → Ask user
